# Отчет по эксперименту 2.4
Сравнение эффективности методов на реальных ML задачах

## 1. Постановка задачи
Сравнить GD, Newton, NLCG, HFN и L-BFGS (`L=10`) по сходимости и времени.

## 2. Функции, данные, параметры, железо
- Оракулы: `LogCoshL2Oracle`, `ExponentialLossL2Oracle`.
- Датасеты: LIBSVM `abalone_scale`, `a9a`.
- Параметры: `λ = 1/m`, `x0 = 0`, line search по сильным условиям Вульфа.
- Методы: GD, Newton, NLCG (PR + restart), HFN, L-BFGS (`L=10`).
- Железо: CPU 4, RAM 16.

## 4. Выводы
- В данном запуске видно, что методы второго порядка и квазиньютоновские методы (Newton/HFN/L-BFGS) достигают низких значений функции за малое число внешних итераций.
- На классификации GD резко проигрывает по времени (десятки секунд против порядка секунд у остальных), что показывает большой разрыв между базовым методом первого порядка и продвинутыми методами.
- Контрпример к тезису «лидер по итерациям = лидер по времени» наблюдается между HFN и L-BFGS: число внешних итераций у HFN меньше, но по времени он сходится (по графику функции `f`) примерно за равное время с L-BFGS.

## 5. Ответы на вопросы 2.4.3
1. Меньше всего внешних итераций делают Newton/HFN/L-BFGS; GD и NLCG требуют заметно больше шагов, особенно на «длинном хвосте» сходимости.  
   Лидер по итерациям не всегда лидер по времени: на текущих графиках HFN имеет меньше внешних итераций, но по времени сходимости функции `f` идет примерно на уровне L-BFGS (без заметного выигрыша по секундам).

2. На «широких» и «высоких» датасетах стабильность сохраняется у всех методов

3. Кривые GD и Newton добавлены на оба графика. Разрыв между GD и продвинутыми методами большой: особенно на классификации GD требует на порядок больше времени, тогда как Newton/HFN/L-BFGS быстро достигают того же уровня качества.
