# Отчет по эксперименту 2.4
**Сравнение эффективности методов на реальных ML задачах**

## 1. Постановка задачи
Сравнить NLCG, HFN и L-BFGS (`L=10`) по сходимости и времени.

## 2. Функции, данные, параметры, железо
- Оракулы: `LogCoshL2Oracle`, `ExponentialLossL2Oracle`.
- Датасеты: LIBSVM `abalone_scale`, `a9a`.
- Параметры: `λ = 1/m`, `x0 = 0`, line search по сильным условиям Вульфа.
- Методы: NLCG (PR + restart), HFN, L-BFGS (`L=10`).
- Железо: CPU `<...>`, RAM `<...>`.

## 4. Выводы
- Лидер по внешним итерациям не всегда лидер по wall-clock времени.
- HFN может выигрывать по итерациям, но проигрывать по времени из-за дорогого внутреннего CG.
- L-BFGS часто дает лучший компромисс “скорость/качество” на разреженных задачах.
