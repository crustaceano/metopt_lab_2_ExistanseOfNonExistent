# Отчет по эксперименту 2.3
Выбор размера истории в методе L-BFGS

## 1. Постановка задачи
Исследовать влияние `L` на скорость сходимости и итоговое время L-BFGS.

## 2. Функции, данные, параметры, железо
- Оракулы: `LogCoshL2Oracle` (регрессия), `ExponentialLossL2Oracle` (классификация).
- Датасеты: LIBSVM `abalone_scale`, `a9a`.
- Параметры: `L ∈ {0,1,5,10,50,100}`, `λ = 1/m`, `x0 = 0`.
- Критерий остановки: относительный по квадрату нормы градиента.
- Железо: AMD Ryzen 5 5600H, RAM 16Gb.

  - итоговое время vs `L`.

## 4. Выводы
- Малые значения `L` дают заметный выигрыш относительно `L=0`.
- После порога по `L` наблюдается плато по итерациям.
- Слишком большой `L` может ухудшать время из-за роста цены итерации.
- Память и арифметика L-BFGS без учета оракула имеют порядок `O(nL)`.
