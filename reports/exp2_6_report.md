# Отчет по эксперименту 2.6
**Оптимизационная точность против качества предсказания**

## 1. Постановка задачи
Проверить, как уменьшение оптимизационной ошибки (значение функции на train и норма градиента) связано с качеством модели на test.

## 2. Функции, данные, параметры, железо
- Train/test split: `80/20`.
- Метод оптимизации: `L-BFGS` (`L=10`, line search по сильным условиям Вульфа).
- Оракулы: `LogCoshL2Oracle` (регрессия), `ExponentialLossL2Oracle` (классификация).
- Датасеты: LIBSVM `abalone_scale`, `a9a`.
- Регуляризация: `lambda = 1/m`.
- Начальная точка: `x0 = 0`.
- Метрики:
  - регрессия: MSE (при необходимости MAE),
  - классификация: Accuracy.
- Железо: AMD Ryzen 5 5600H, RAM 16Gb.

## 4. Выводы
1. **Точка насыщения test-метрики.** По текущим кривым и дополнительному прогону точка насыщения для регрессии стабильно около `11`-й итерации для `lambda = 1/m, 1/(10m), 1/(100m)`. Для классификации насыщение наступает очень рано, около `2`-й итерации для всех трех `lambda`. Это согласуется с визуальной картиной: train-кривые продолжают убывать, но вклад в test после этих итераций минимален.

2. **Доля «бесполезной» оптимизации и выбор `eps`.** После точки насыщения основное улучшение идет только в train-цели, поэтому финальная часть траектории практически «бесполезна» для test-метрики. По прогону с `max_iter=30` доля времени после точки насыщения составила:
   - регрессия: примерно `64-67%`,
   - классификация: примерно `93-94%`.
   Это подтверждает, что ранний останов критичен, особенно в классификации. Для практического раннего останова на этих задачах разумно брать пороги:
   - `eps ~ 10^-2` (регрессия),
   - `eps ~ 10^-1` (классификация),
   что обычно останавливает оптимизацию близко к началу плато качества.

3. **Влияние `lambda` на точку насыщения.** Проведен отдельный sweep: `lambda = 1/m`, `1/(10m)`, `1/(100m)`.
   - В регрессии точка насыщения по итерациям не сдвинулась (около `11`), но лучшее test MSE немного улучшилось при меньших `lambda` (`~5.47 -> ~5.26`).
   - В классификации точка насыщения также не сдвинулась (около `2`), а best test Accuracy осталась на одном уровне (`~0.8558`).
   - Явного роста разрыва train/test при уменьшении `lambda` в этом диапазоне не наблюдается; эффект скорее в небольшом улучшении test MSE для регрессии и почти неизменной test Accuracy для классификации.
